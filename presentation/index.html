Phase A: Ingestion Quality Control

Dual-Path Write & Stream DLQ

🛡️

Strict Pydantic Validation

Every application payload is parsed against a strong Pydantic model (`schemas.py`) before operational caching.

  • Corrupt Detection: Blocks invalid types (e.g. string instead of float, missing IDs).
  • Idempotency: Valid entries are upserted into PostgreSQL cache using `ON CONFLICT`.
🚨

Stream Dead-Letter Queue

Failures must not break the streaming loop. Invalid payloads are isolated for easy diagnostics.

  • Dual-Storage DLQ: Writes malformed records to Postgres `stream_dlq` and `corrupt_events.jsonl` files.
  • Detailed Audit Trail: Captures raw validation error traces right next to the event payload.
Phase B: Orchestrated Ingestion

Scheduled Partner Data Ingestion

🗓️

Airflow DAG Orchestration

A scheduled batch engine pulls financial transaction files delivered by third-party partners.

  • File Sensors: DAG pauses until transaction data directories populate on disk.
  • Automated Retries: Set up tasks with robust backoffs to survive temporary db locking.
📂

Analytical Batch Database

Successfully processed transactions are loaded to Postgres relational tables.

  • Transaction Modeling: Columns partitioned, validated, and normalized during upsert.
  • SQL UPSERT: Safely runs multiple times without creating duplicate records.

Phase B: Resilience & Archival

Data Archival & Batch DLQ Isolation

🗜️

Partner Log Compression Archive

Once files are scanned, they are moved to a local cold storage archive to minimize ingestion footprint.

  • Gzip Compression: Transferred files are compressed as `*.txt.gz` and timestamped.
  • Regulatory Auditing: Preserves original CSV layouts without wasting active host storage.
⚡

Batch Row Isolation & DLQ

Individual corrupted CSV lines are isolated so they don't block valid adjacent rows.

  • Fault-Tolerant Parsing: Catches schema drift or bad characters during CSV iteration.
  • Corrupt Logs: Offloads corrupted rows to Postgres `batch_dlq` and a `corrupt_rows.jsonl` file.

Summary & Design Philosophy

Engineering Highlights & Key Takeaways

An enterprise-grade template focusing on structural reliability, data isolation, and comprehensive pipeline visibility.

Storage Optimization

  • PG Operational Cache: Provides rel. aggregations and saves local dev memory footprint.
  • Hourly Partitions: Optimizes downstream analytical query scanning by over 95%.

Production Resilience

  • Pydantic schemas: Assures high-quality ingestion streams.
  • Isolated DLQs: Directs faulty rows to DB / JSONL tables, preventing pipeline stalls.

Developer Velocity

  • 1-Step Orchestration: Single Docker compose up commands bootstrap the full flow.
  • Offline Kinesis: Pinned LocalStack setup executes completely offline.
